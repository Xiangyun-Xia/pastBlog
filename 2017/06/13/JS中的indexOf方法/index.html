  • String 类型的使用

    这里 0 和 3 分别是 o 和 n 在字符串中出现的位置。起始下标是 0。而 -1 代表未匹配。

    可以发现 indexOf 会做简单的类型转换，把数字转换成字符串 ‘2’ 然后再执行。

  • Number 类型的使用

    number类型有没有 indexOf 方法，因为不会进行隐式转换，如果要使用，需要把number类型转化为string类型。

  • Array 类型的使用

    这个便是我今天所碰到的问题了。

    以上四个用例足以说明问题。

    1. arr.indexOf('orange')输出 0 因为 ‘orange’ 是数组的第 0 个元素，匹配到并返回下标。
    2. arr.indexOf('o') 输出 -1 因为此方法不会在每一个元素的基础上再次执行 indexOf 匹配。
    3. arr.indexOf('2016') 输出 1 因为此方法从头匹配直到匹配到时返回第一个数组元素的下表，而不是返回全部匹配的下标。
    4. arr.indexOf(2016)输出 -1 注意：这里不会做隐式类型转换。

    而在我今天的问题中，之所以始终返回-1，是因为我判断对象是否存在于数组中，而对象存储在堆中，它指向堆中的一块内存地址，所以除非内存地址指向相同，否则比较出来都返回-1。
